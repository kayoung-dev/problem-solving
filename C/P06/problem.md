# 후위 표기식 계산 (Postfix Evaluation)

## 문제 설명
일반적인 수식은 `2 + 3 * 4`처럼 연산자가 피연산자 사이에 들어가는 **중위 표기식(Infix Notation)**을 사용합니다.
하지만 컴퓨터가 수식을 계산할 때는 연산자가 피연산자 뒤에 오는 **후위 표기식(Postfix Notation)**, 일명 '역폴란드 표기법(Reverse Polish Notation)'이 훨씬 처리하기 쉽습니다.

후위 표기식의 계산 규칙은 다음과 같습니다:
1. 수식을 앞에서부터 읽으면서 숫자를 만나면 **스택(Stack)**에 넣습니다 (`push`).
2. 연산자(`+`, `-`, `*`, `/`)를 만나면 스택에서 두 개의 숫자를 꺼냅니다 (`pop`).
   * 먼저 꺼낸 수가 연산자의 오른쪽 피연산자, 나중에 꺼낸 수가 왼쪽 피연산자가 됩니다.
3. 두 수에 대해 해당 연산을 수행하고, 그 결과를 다시 스택에 넣습니다.
4. 수식이 끝나면 스택에 남은 마지막 숫자가 최종 결과입니다.

문자열 배열 형태의 후위 표기식이 주어졌을 때, 그 계산 결과를 구하는 프로그램을 작성하세요.
(나눗셈은 정수 나눗셈으로, 소수점 이하는 버립니다. 예를 들어 `3 / 2`는 `1`입니다.)

---

## 입력 형식 (Input Format)
* 첫 번째 줄에 연산(토큰)의 개수 $N$이 주어집니다. ($1 \le N \le 100$)
* 두 번째 줄에 후위 표기식이 공백으로 구분되어 주어집니다.
    * 피연산자는 정수이며, 연산자는 `+`, `-`, `*`, `/` 중 하나입니다.
    * 주어지는 식은 항상 유효한 후위 표기식임이 보장됩니다.
    * 0으로 나누는 경우는 없습니다.

## 출력 형식 (Output Format)
* 수식의 계산 결과를 정수로 출력합니다.

---
## 입출력 예시 (Sample I/O)

### 예시 1
**Input:**
```
5
2 3 + 5 *
```
**Output:**
```
25
```
* 계산 과정:
  1. `2`, `3`을 스택에 넣음. (Stack: `[2, 3]`)
  2. `+`를 만남 -> `3`, `2`를 꺼내 `2 + 3 = 5` 계산 후 저장. (Stack: `[5]`)
  3. `5`를 스택에 넣음. (Stack: `[5, 5]`)
  4. `*`를 만남 -> `5`, `5`를 꺼내 `5 * 5 = 25` 계산 후 저장. (Stack: `[25]`)
  5. 최종 결과: 25

### 예시 2
**Input:**
```
5
4 13 5 / +
```
**Output:**
```
6
```
* `13 / 5`는 정수 나눗셈으로 `2`가 됩니다. `4 + 2 = 6`.
