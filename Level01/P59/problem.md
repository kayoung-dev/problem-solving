# 실시간 네트워크 트래픽 분석

## 문제 설명
네트워크 관리자인 당신은 서버로 유입되는 패킷의 크기를 실시간으로 모니터링하여 네트워크의 상태를 파악하려 합니다. 급격한 트래픽 변화를 감지하기 위해, 최근 **K초 동안 유입된 패킷 크기의 평균**을 지속적으로 계산해야 합니다.

시간의 흐름에 따라 기록된 N개의 패킷 크기 데이터와 분석할 시간 간격 K가 주어질 때, 매초 계산되는 이동 평균(Moving Average)의 **정수 부분**을 순서대로 출력하는 프로그램을 작성하세요.

- 공학적 설계 규칙
1. 새로운 패킷 데이터가 들어오면 큐의 맨 뒤(Rear)에 추가합니다.
2. 큐에 담긴 데이터의 개수가 K개를 초과하면, 가장 오래된 데이터(Front)를 큐에서 제거합니다.
3. 매 순간 큐에 담긴 **K개 데이터의 합계를 K로 나눈 평균값(소수점 버림)** 을 계산합니다.
4. 전체 데이터 N개 중, 처음 K개의 데이터가 모여 첫 번째 평균을 계산할 수 있는 시점부터 출력을 시작합니다.

---

## 입력 형식 (Input Format)
- 첫 번째 줄에 전체 데이터의 개수 N과 시간 간격 K가 공백으로 구분되어 주어집니다. (1 <= K <= N <= 100,000)
- 두 번째 줄에 N개의 패킷 크기(0 이상의 정수)가 시간 순서대로 주어집니다.

## 출력 형식 (Output Format)
- 각 시점의 이동 평균값(정수)을 순서대로 공백으로 구분하여 출력합니다.
- 출력되는 값의 개수는 N - K + 1개가 됩니다.

---

## 입출력 예시 (Sample I/O)

### 예시 1
**Input:**
```
6 3
10 20 30 40 50 60
```

**Output:**
```
20 30 40 50
```

- 윈도우 크기(K)가 3입니다.
- [10, 20, 30] 유입 -> 평균: (10+20+30)/3 = **20**
- 40 유입, 10 제거 -> [20, 30, 40] -> 평균: (20+30+40)/3 = **30**
- 50 유입, 20 제거 -> [30, 40, 50] -> 평균: (30+40+50)/3 = **40**
- 60 유입, 30 제거 -> [40, 50, 60] -> 평균: (40+50+60)/3 = **50**

### 예시 2
**Input:**
```
5 2
10 15 20 25 30
```

**Output:**
```
12 17 22 27
```

### 예시 3 
**Input:**
```
7 4
10 20 30 40 50 60 70
```

**Output:**
```
25 35 45 55
```

- 윈도우 크기(K)가 4입니다. 4개가 모여야 첫 평균을 냅니다.
- [10, 20, 30, 40] 유입 -> 합계 100, 평균: 100/4 = **25**
- 50 유입, 10 제거 -> [20, 30, 40, 50] -> 합계 140, 평균: 140/4 = **35**
- 60 유입, 20 제거 -> [30, 40, 50, 60] -> 합계 180, 평균: 180/4 = **45**
- 70 유입, 30 제거 -> [40, 50, 60, 70] -> 합계 220, 평균: 220/4 = **55**
